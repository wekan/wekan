2024-11-25

  • At morning, SaaS hosting was moved to different server, because there was notification that there will be 6 hours downtime at network of server where hosting was previously, because they are making some changes to improve network quality. Moving had minimal downtime.
  • Later at evening, there was some downtime.
  • 20:15 Fixing done, all services back online.
  • Root cause:
  • 1) For Snap, reboot of server made them work.
  • 2) For RocketChat, lxc containers did not have any network connectivity. Fix was these commands from this webpage: https://discuss.linuxcontainers.org/t/containers-do-not-have-outgoing-internet-access/10844/3
for ipt in iptables iptables-legacy ip6tables ip6tables-legacy; do $ipt --flush; $ipt --flush -t nat; $ipt --delete-chain; $ipt --delete-chain -t nat; $ipt -P FORWARD ACCEPT; $ipt -P INPUT ACCEPT; $ipt -P OUTPUT ACCEPT; done

systemctl reload snap.lxd.daemon
